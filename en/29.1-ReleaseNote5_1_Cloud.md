# v5.1[](#v5-1 "Link to this heading")

**Features summary**

Version 5.1 incorporates Q-flow in the cloud, supporting the use of multiple tenants, different time zones and a new web site “Client Registration”, used to create and manage tenants, purchase and manage Q-flow Cloud licenses and get the access links for the created tenants.

**Q-flow Cloud**

The product is incorporated to the Cloud through the service of Microsoft Azure. This provides various improvements, such as accessibility, since unlike the OnPremise version, Cloud allows you to access Q-flow at any time and place, from any device.

Q-flow Cloud now incorporates new concepts for cloud licensing:

> *   Q-points: points that represent the effort that running a process takes. These are consumed when executing steps in a flow. Each step has a certain amount of Q-points assigned to it. In some cases this amount can vary over time.
>     
> *   Storage: amount of storage capacity that the tenant has available, represented in gigabytes. Elements such as attachments are the ones that affect this amount.
>     

## Client Registration[](#client-registration "Link to this heading")

Site to create new tenants by subscribing to the plan that better fits your needs (monthly, biannually, annually). It also allows you to edit or delete them, as well as showing a unified view of all of your tenants and access links to the different sites.

## Web SAM[](#sam-web "Link to this heading")

### Home page update[](#id1 "Link to this heading")

On the SAM site, indicators of the current consumption of storage (amount in gigabytes of storage that the tenant has available. Elements such as attachments affect this amount), Q-points and users of the tenant have been added. When any of these values surpasses the limit, or when the user does not have an enabled license, the indicator will turn red. The values of these indicators is cumulative, this means thet if you have several licenses their resources will be accumulated on the indicators.
