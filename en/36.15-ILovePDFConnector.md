# ILovePDF[](#ilovepdf "Link to this heading")

.offset-anchor { display: block; position: relative; top: -2em; visibility: hidden; }

## Introduction[](#introduccion "Link to this heading")

The purpose of this manual is to detail how to configure and integrate Qflow with the ILovePDF connector, describing each of the actions it provides for handling documents or PDF files and how to incorporate them into the platform’s processes.

The connector has the following action:

*   [Convert Office File](#id1)
    
*   [Merge PDF Files](#id2)
    
*   [Convert Image to PDF](#id3)
    

## Prerequisites[](#prerrequisitos "Link to this heading")

A **ILovePDF** account and a **public API Key** with conversion permissions are required.

## Application Parameter[](#parametro-de-aplicacion "Link to this heading")

To establish the connection, an **application parameter** is required (see [Application Parameters](15-QflowDesign.html#parametros-de-aplicacion)) to enable communication between Qflow and ILovePDF. This parameter is used to configure the connector within a Qflow Service Task (see [Connector Configuration from a Service Task](35-ConnectorParameterConfig.html)).

To create an ILovePDF application parameter, the following steps are required, detailed below.

### Application parameter using **Public API Key**[](#parametro-de-aplicacion-utilizando-api-key-publica "Link to this heading")

This type of application parameter requires the following:

*   **API Key**: Public key generated from the ILovePDF dashboard.
    

To obtain it, follow these steps:

1.  Go to the ILovePDF website ([https://www.iloveapi.com/](https://www.iloveapi.com/)) and create an account, or use an existing one.
    
2.  Go to **Projects → API Keys** and copy the public key.
    
    [![_images/ApiKeyDashboard.png](_images/ApiKeyDashboard.png)](_images/ApiKeyDashboard.png)
    
    Fig. 863 ILovePDF panel with **Public API Key** section.[](#id4 "Link to this image")
    
    It is not mandatory to use a different API Key per process. However, it is recommended to generate one key per process to facilitate management, usage monitoring, and security. You can generate new keys from the following panel:
    
    [![_images/ApiKeyPanel.png](_images/ApiKeyPanel.png)](_images/ApiKeyPanel.png)
    
    Fig. 864 **Public API Key** available to copy, and a link to create new projects.[](#id5 "Link to this image")
    
    If you want to create a new key, simply assign it a name and save it. It will be generated in the panel shown above.
    
    [![_images/NewApiKEY.png](_images/NewApiKEY.png)](_images/NewApiKEY.png)
    
    Fig. 865 Panel to create new API Keys[](#id6 "Link to this image")
    
3.  In **Qflow Design**, select the connector and configure the obtained **Public API Key** as the connection parameter.
    
    [![_images/DesignParametro.png](_images/DesignParametro.png)](_images/DesignParametro.png)
    
    Fig. 866 Data added in the application parameter[](#id7 "Link to this image")
    

## Actions[](#acciones "Link to this heading")

With ILovePDF, the following action can be performed:

### Convert Office File[](#id1 "Link to this heading")

This action allows converting Office documents into a single PDF file.

|     |     |
| --- | --- |Table 66 Inputs[](#id8 "Link to this table")  
| Input | Description |
| --- | --- |
| File to convert | **Required.** Enter the name of the document-type data item that contains the Office file you want to convert to PDF. The file size must not exceed 50 MB. |
| Name of the converted PDF file | Enter the name of the converted PDF file. You cannot use characters not allowed by Windows, such as: ? " \| \* / \\ < >. |

|     |     |
| --- | --- |Table 67 Outputs[](#id9 "Link to this table")  
| Output | Description |
| --- | --- |
| Converted PDF file | Enter a document-type data item that will contain the final PDF file resulting from converting the selected file. |

### Merge PDF files[](#id2 "Link to this heading")

This action allows you to merge multiple PDF documents into a single resulting file.

|     |     |
| --- | --- |Table 68 Inputs[](#id10 "Link to this table")  
| Input | Description |
| --- | --- |
| Files to merge | **Required.** Enter the name or names of the document-type data items that will contain the PDF files to merge. The total size of the files must not exceed 50 MB. |
| Name of the resulting file | Enter the name of the resulting merged PDF file. You cannot use characters not allowed by Windows, such as: ? " \| \* / \\ < >. If left blank, result.pdf will be used. |

|     |     |
| --- | --- |Table 69 Outputs[](#id11 "Link to this table")  
| Output | Description |
| --- | --- |
| Resulting PDF file | Enter a document-type data item that will contain the final PDF file produced by merging the selected files. |

### Convert image to PDF[](#id3 "Link to this heading")

This action allows you to convert a single image into a PDF file.

|     |     |
| --- | --- |Table 70 Inputs[](#id12 "Link to this table")  
| Input | Description |
| --- | --- |
| Image to convert | **Required.** Enter the name of the document-type data item that will contain the image to convert. The total size must not exceed 50 MB. Supported extensions: JPG, JPEG, PNG, TIF, TIFF. |
| Name of the converted PDF file | Enter the name that the generated PDF file will have. You cannot use characters not allowed by Windows, such as: ? " \| \* / \\ < >. |

|     |     |
| --- | --- |Table 71 Outputs[](#id13 "Link to this table")  
| Output | Description |
| --- | --- |
| Converted image | Enter a document-type data item that will contain the final PDF file generated by converting the selected image. |
