# v6.0[](#v6-0 "Link to this heading")

La versión 6.0 de Qflow incorpora nuevas funcionalidades que mejoran la experiencia del usuario, amplían las capacidades de diseño y facilitan la integración con herramientas externas.

## Diseñador de formularios[](#disenador-de-formularios "Link to this heading")

Se incorpora una nueva herramienta de diseño de formularios dentro del módulo **Design**, que permite construir formularios personalizados de manera visual e intuitiva.

*   Nuevo editor drag & drop para la creación de formularios.
    
*   Posibilidad de reordenar datos y roles libremente dentro del formulario.
    
*   Los roles ya no necesitan estar agrupados, ahora pueden ubicarse en cualquier parte del formulario. Lo mismo aplica para la bitácora de comentarios.
    
*   Estética moderna y adaptable.
    
*   Soporte para agregar títulos y mensajes informativos.
    
*   Formularios completamente integrables en los procesos de Qflow.

## Nueva acción: agregar adjuntos a incidencia en JIRA[](#nueva-accion-agregar-adjuntos-a-incidencia-en-jira "Link to this heading")

Se incorpora una nueva acción para el conector con JIRA. Este permite enviar archivos adjuntos directamente desde Qflow a un issue específico de JIRA, mejorando la integración con flujos de trabajo en herramientas externas.

Más información en la [documentación del conector de JIRA](36.6-JiraCloudConnector.md).

[![JIRA](_images/JIRA.png)](_images/JIRA.png)

Figura 2 JIRA[](#id2 "Link to this image")

## Eliminar proceso finalizado[](#eliminar-proceso-finalizado "Link to this heading")

Ahora es posible eliminar procesos que ya se encuentren finalizados, lo que facilita la gestión y limpieza de instancias innecesarias o creadas por error. A su vez, se libera espacio de almacenamiento.

[![Eliminar procesos finalizados](_images/DeleteFlows.png)](_images/DeleteFlows.png)

Figura 3 Eliminar procesos finalizados[](#id3 "Link to this image")

## Nuevo modelo de visibilidad de datos[](#nuevo-modelo-de-visibilidad-de-datos "Link to this heading")

Se redefine el concepto de visibilidad de los datos dentro de los formularios, con un enfoque más claro y estructurado.

*   Los nuevos estados posibles son: **Ausente**, **Solo lectura**, **Editable** y **Requerido**.
    
*   Para mantener compatibilidad con configuraciones anteriores (por ejemplo, «solo lectura editable»), se agrega una opción adicional: «**Permite modificar el valor por código o dependencia**», que flexibiliza el comportamiento según lógica de negocio.
