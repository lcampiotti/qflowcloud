# Slack[](#slack "Link to this heading")

## Introducción[](#introduccion "Link to this heading")

El propósito de este manual es detallar cómo configurar e integrar Qflow con Slack para permitir la interacción entre ambas plataformas, permitiendo enviar y responder a mensajes de Slack desde Qflow.

Slack cuenta con las siguientes acciones:

*   [Enviar mensaje (Slack App)](#id1)
    
*   [Enviar mensaje (Webhook)](#id2)
    

## Prerrequisitos[](#prerrequisitos "Link to this heading")

Es necesario contar con una cuenta de Slack y un espacio de trabajo creado en la plataforma. Esto puede hacerse [desde aquí](https://slack.com/get-started#/create).

## Parámetro de aplicación[](#parametro-de-aplicacion "Link to this heading")

Para entablar la conexión es necesario contar con al menos un **parámetro de aplicación** (ver [Parámetros de aplicación](15-QflowDesign.md#parametros-de-aplicacion)) que permita establecer la comunicación entre Qflow y Slack. Este parámetro puede ser creado desde la configuración de la tarea de servicio en Qflow. (ver [Configuración de Conectores desde una Tarea de Servicio](35-ConnectorParameterConfig.md))

Para crear un parámetro de aplicación de Slack, se requieren los siguientes pasos que se detallan a continuación. Para establecer una conexión únicamente va a ser necesario configurar uno de ellos.

### Parámetro de aplicación utilizando Token de Autenticación OAuth[](#parametro-de-aplicacion-utilizando-token-de-autenticacion-oauth "Link to this heading")

Este tipo de parámetro de aplicación requiere el siguiente parámetro:

*   **Token de Autenticación OAuth**: Es un token de autenticación que permite a Qflow conectarse a Slack como un Bot, simulando ser un usuario y realizando operaciones en nombre de la aplicación.
    

Para obtenerlo, se deben seguir los siguientes pasos:

1.  Acceder a la [sección de aplicaciones de Slack](https://api.slack.com/apps). Aquí se crea una nueva aplicación al seleccionar Create New App.
    
    [![_images/OAuth1.png](_images/OAuth1.png)](_images/OAuth1.png)
    
    Figura 792 Botón Create New App en la sección de aplicaciones de Slack[](#id3 "Link to this image")
    
2.  Seleccionar la opción From scratch para especificar que se está creando la aplicación desde cero.
    
    [![_images/OAuth2.png](_images/OAuth2.png)](_images/OAuth2.png)
    
    Figura 793 Vista de selección de tipo de creación de aplicación en Slack[](#id4 "Link to this image")
    
3.  Escribir el nombre de la aplicación y seleccionar el workspace al que pertenecerá, luego seleccionar Create App. En el ejemplo de la imagen a continuación, se muestra que la aplicación se llama Qflow y pertenece al workspace Nombre de empresa.
    
    [![_images/OAuth3.png](_images/OAuth3.png)](_images/OAuth3.png)
    
    Figura 794 Vista de creación de aplicación en Slack[](#id5 "Link to this image")
    
4.  Seleccionar la opción OAuth & Permissions en el menú de la izquierda.
    
    [![_images/OAuth4.png](_images/OAuth4.png)](_images/OAuth4.png)
    
    Figura 795 OAuth & Permissions marcado en el menú de la aplicación de Slack[](#id6 "Link to this image")
    
5.  Dentro de OAuth & Permissions, en la sección Scopes, seleccionar la opción Add an OAuth Scope para agregar un permiso a la aplicación.
    
    [![_images/OAuth5.png](_images/OAuth5.png)](_images/OAuth5.png)
    
    Figura 796 Add an OAuth Scope marcado en la sección de permisos de la aplicación de Slack[](#id7 "Link to this image")
    
6.  Agregar los permisos channels:read y chat:write, los cuales permiten leer la información básica de los canales y escribir mensajes en ellos, respectivamente.
    
    [![_images/OAuth6.png](_images/OAuth6.png)](_images/OAuth6.png)
    
    Figura 797 Permisos channels:read y chat:write agregados a la sección de permisos de la aplicación de Slack[](#id8 "Link to this image")
    
7.  Dentro de OAuth & Permissions, en la sección OAuth Tokens for Your Workspace, seleccionar Install to Workspace
    
    [![_images/OAuth7.png](_images/OAuth7.png)](_images/OAuth7.png)
    
    Figura 798 Install to Workspace marcado en la sección de tokens de autenticación de la aplicación de Slack[](#id9 "Link to this image")
    
8.  Confirmar la instalación de la aplicación en el workspace.
    
    [![_images/OAuth8.png](_images/OAuth8.png)](_images/OAuth8.png)
    
    Figura 799 Permitir marcado en la confirmación de permisos de acceso de la aplicación de Slack[](#id10 "Link to this image")
    
9.  Una vez confirmados los permisos, Slack muestra el Bot User OAuth Token, que es el token de autenticación necesario para configurar el parámetro de aplicación en Qflow. Con esto finalizan los pasos necesarios en la página Slack API.
    
    [![_images/OAuth9.png](_images/OAuth9.png)](_images/OAuth9.png)
    
    Figura 800 Bot User OAuth Token marcado en la sección de tokens de autenticación de la aplicación de Slack[](#id11 "Link to this image")
    
10.  En Qflow Design, agregar el Bot User OAuth Token en el parámetro Token de autenticación OAuth
    
    [![_images/OAuth10.png](_images/OAuth10.png)](_images/OAuth10.png)
    
    Figura 801 Token de autenticación OAuth agregado en Qflow[](#id12 "Link to this image")
    
11.  Desde el chat de Slack, es necesario agregar la aplicación a cada canal de Slack al que se desea permitir enviar mensajes. Se debe seleccionar la opción Añadir aplicaciones a este canal en el canal deseado.
    
    [![_images/OAuth11.png](_images/OAuth11.png)](_images/OAuth11.png)
    
    Figura 802 Añadir aplicaciones a este canal seleccionado como acción en el chat de Slack[](#id13 "Link to this image")
    
12.  Buscar la aplicación Qflow y añadirla al canal seleccionado. Para esto, se debe de seleccionar Añadir.
    
    [![_images/OAuth12.png](_images/OAuth12.png)](_images/OAuth12.png)
    
    Figura 803 Añadir seleccionado en la ventana de añadir aplicaciones a un canal de de Slack[](#id14 "Link to this image")
    

Una vez completados los pasos, será posible enviar mensajes desde Qflow a Slack utilizando Slack App.

### Parámetro de aplicación utilizando Webhook[](#parametro-de-aplicacion-utilizando-webhook "Link to this heading")

Este tipo de parámetro de aplicación requiere el siguiente parámetro:

*   **Vínculo del Webhook**: Es un vínculo que representa un canal de comunicación de Slack, permitiendo a Qflow enviar mensajes dentro de este canal.
    

Para obtenerlo, se deben seguir los siguientes pasos:

1.  Realizar los pasos del 1 al 3 del apartado anterior. (ver [Parámetro de aplicación utilizando Token de Autenticación OAuth](#parametro-de-aplicacion-utilizando-token-de-autenticacion-oauth))
    
2.  Seleccionar la opción Incoming Webhooks.
    
    [![_images/Webhook1.png](_images/Webhook1.png)](_images/Webhook1.png)
    
    Figura 804 Incoming Webhooks seleccionado en el menú de agregar características y funcionalidades de la aplicación de Slack[](#id15 "Link to this image")
    
3.  Activar los Incoming Webhooks para poder utilizarlos.
    
    [![_images/Webhook2.png](_images/Webhook2.png)](_images/Webhook2.png)
    
    Figura 805 Activate Incoming Webhooks seleccionado para activar la funcionalidad de Webhooks en la aplicación de Slack[](#id16 "Link to this image")
    
4.  Una vez activados los Incoming Webhooks, seleccionar la opción Add New Webhook to Workspace. Cada webhook creado estará asociado a un canal específico de Slack.
    
    [![_images/Webhook3.png](_images/Webhook3.png)](_images/Webhook3.png)
    
    Figura 806 Add New Webhook to Workspace seleccionado en la vista Incoming Webhooks de la aplicación de Slack[](#id17 "Link to this image")
    
5.  Seleccionar el canal al que se desea asociar el webhook y luego seleccionar Permitir.
    
    [![_images/Webhook4.png](_images/Webhook4.png)](_images/Webhook4.png)
    
    Figura 807 Permitir seleccionado en la vista de selección de canal para asociar un webhook en la aplicación de Slack[](#id18 "Link to this image")
    
6.  Slack mostrará el Webhook URL del webhook creado para el canal seleccionado, necesario para configurar el parámetro de aplicación en Qflow. Con esto finalizan los pasos necesarios en la página Slack API.
    
    [![_images/Webhook5.png](_images/Webhook5.png)](_images/Webhook5.png)
    
    Figura 808 Webhook URL mostrado en la vista de creación de webhook en la aplicación de Slack[](#id19 "Link to this image")
    
7.  En Qflow Design, agregar el Webhook URL en el parámetro Vínculo del Webhook
    
    [![_images/Webhook6.png](_images/Webhook6.png)](_images/Webhook6.png)
    
    Figura 809 Vínculo del Webhook agregado en Qflow[](#id20 "Link to this image")
    

Una vez completados los pasos, será posible enviar mensajes desde Qflow a Slack utilizando Webhook.

## Acciones[](#acciones "Link to this heading")

Se pueden realizar las siguientes acciones con Slack:

### Enviar mensaje (Slack App)[](#id1 "Link to this heading")

Esta acción permite enviar un mensaje directamente a un canal de Slack. Se debe proporcionar el nombre del canal (sin el “#”) y luego el mensaje. Si se desea responder a otro mensaje, también se puede agregar su identificador. Después de enviar el mensaje, se recibe un número que identifica el mensaje, lo que permite responder a él más adelante.

|     |     |
| --- | --- |Tabla 6 Entradas[](#id21 "Link to this table")  
| Entrada | Descripción |
| --- | --- |
| Nombre del canal | **Requerido.** El nombre del canal de Slack al que se desea enviar el mensaje. (Sin “#” al principio) |
| Mensaje | **Requerido.** Texto del mensaje que se desea enviar por Slack. |
| El identificador del mensaje al que se desea responder. | Se obtiene al crear un mensaje desde Qflow. Si no se desea responder a un mensaje, este parámetro se puede dejar en blanco. |

|     |     |
| --- | --- |Tabla 7 Salidas[](#id22 "Link to this table")  
| Salida | Descripción |
| --- | --- |
| Identificador del mensaje | El número que identifica el mensaje que se acaba de enviar. Se puede usar este número para responder a este mensaje. |

### Enviar mensaje (Webhook)[](#id2 "Link to this heading")

Esta opción permite enviar mensajes a un canal de Slack, pero de forma más limitada. El canal al que envías el mensaje está determinado por el Webhook que se utiliza, ya que cada Webhook está asociado a un canal específico. No es posible responder a mensajes directamente con esta opción, ya que no se puede especificar el identificador del mensaje al que se desea responder.

|     |     |
| --- | --- |Tabla 8 Entradas[](#id23 "Link to this table")  
| Entrada | Descripción |
| --- | --- |
| Mensaje | **Requerido.** Texto del mensaje que se desea enviar por Slack. |
